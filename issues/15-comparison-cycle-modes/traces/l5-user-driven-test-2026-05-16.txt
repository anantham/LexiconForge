3 BUGS FOUND DURING L5 (none caught by L2 unit tests because tests mocked
lookupWord and didn't exercise the actual prop-flow + ESM-export caching):

  Bug A — LLM offset hallucination
    Symptom: alignment returned 0 pairs. validateAlignment dropped them all
             because the LLM gave correct source words but incorrect char offsets
             (off by ~10 chars in target slice).
    Fix: rewrote validateAlignment to RECOMPUTE offsets via indexOf rather
         than trust LLM's offsets. Source/target words are still LLM's choice;
         just the char positions are recovered.

  Bug B — perWordTranslation cached empty glossary results
    Symptom: hovering a pair the first time (with empty glossary) cached [].
             Adding glossary entries later didn't surface — the cache short-
             circuited the new lookup.
    Fix: don't cache glossary results (lookup is in-memory list filter, free).
         Network-provider results still cache.

  Bug C — Component-level `fetched` flag blocked re-fetches on prop change
    Symptom: even with cache fix, the WordPairToken's `if (fetched) return`
             early-exit prevented re-running lookupWord when glossary prop
             changed after the first hover.
    Fix: dropped the `fetched` flag; rely on perWordTranslation's own cache
         for dedup. Hover always fires, dedup happens at service layer.
